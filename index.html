1. 页面总览

左侧导航是系统的主要工作区。日常使用通常从“新建反馈”开始，后续在“历史记录”“学生汇总”“成绩分析”和“数据概览”里回看、分析和导出。

页面主要功能适合什么时候看
新建反馈记录班级整体学情、个人反馈、授课内容和作业情况。每节课结束后。
历史记录按班级、科目、月份和关键词筛选过往反馈。复盘某节课、查找某位学生记录时。
学生汇总查看学生累计反馈、标签、作业完成率和个人时间线。准备家长沟通、阶段反馈时。
成绩分析导入考试成绩，生成学科对比、均分、及格率、排名和趋势。月考、模考、单元测后。
数据概览查看反馈总量、科目分布、班级覆盖和标签分布。检查记录是否持续、是否有学生被遗漏时。
系统设置维护体系、班级、学生、科目、内容库、标签和备份。首次配置、学期调整、备份恢复时。

2. 新建反馈

这是课后最高频的录入页面。顶部填写日期、班级、科目和授课内容；中间选择班级整体学情；下方给需要单独记录的学生填写专注度、作业完成度和简短反馈。

  • 班级整体反馈：用于记录整节课的共性情况，例如互动、纪律、掌握程度。
  • 个人反馈：用于记录某个学生的突出表现或需要关注的问题。
  • 授课内容库：提前维护常用单元、知识点和作业，录入时可直接选择。

3. 历史记录

历史记录是所有反馈的检索入口。可以按班级、科目、月份、关键词筛选，也可以打开记录详情进行查看、编辑或删除。

如果要找“某个学生最近几次课的表现”，优先用学生姓名或关键词搜索；如果要复盘一个阶段，优先用月份和科目筛选。

4. 学生汇总

学生汇总把分散的课堂记录聚合到学生维度。这里重点看三类信息：记录次数、作业完成率、标签变化。

  • 记录次数：反映这个学生被关注和记录的频率。
  • 作业完成率：来自每次反馈中的作业完成情况。
  • 学生标签：包括手动标签和系统根据反馈、成绩生成的自动标签。

5. 成绩分析

成绩分析用于处理考试后的班级和个人数据。导入成绩后，系统会生成学科对比表、均分柱状图、学生明细、需关注学生和个人趋势。

  • 学科对比表：看最高分、最低分、均分、及格率。
  • 学生明细表：看排名、各科分数和不及格科目。
  • 个人趋势：看某位学生跨考试的总分、排名和各科变化。

6. 数据概览

数据概览用于看“系统里已经沉淀了多少数据”。常用指标包括累计反馈数、覆盖科目数、班级数、学生数、各科记录分布和标签分布。

如果某个科目记录明显偏少，说明这段时间该科反馈沉淀不足；如果“需关注”类标签集中出现，适合进一步回到学生汇总查看具体学生。

7. 系统设置

系统设置用于维护基础资料。首次使用时主要完成体系选择、班级创建、学生名单导入、科目设置和授课内容库配置。日常使用中，最重要的是定期导出 JSON 备份。

8. 专有名词

名词含义
体系指高考体系或国际考试体系，会影响默认科目、标签和首页。
内容库老师预先维护的授课单元、知识点和作业库。
反馈记录一次课后保存下来的班级学情和学生个人反馈。
标签用于概括学生状态的短词，例如“进步明显”“需复盘错题”。
自动标签系统根据反馈、作业、成绩等规则自动生成的标签。
JSON 备份完整数据快照，包含反馈、成绩、班级、学生和设置。
学期存档学期末保存当前数据为只读档案，再开始新学期。
